Multi-Robot ROS2 + Gazebo Demo on OpenShift

This is the Gazebo simulation pod. Multiple Nav2 navigation pods run independently, each controlling one TurtleBot3 robot, connected via Zenoh.

View the Gazebo Simulation

Click the button below to open the Gazebo GUI in your browser via noVNC. You will see 2 TurtleBot3 Waffle robots in the sandbox world.

Open Gazebo Visualization (noVNC)

What is Running

  • This pod (gazebo-sim): Gazebo Harmonic, robot spawning, VNC visualization
  • robot-nav-robot-1: Nav2 full navigation stack for robot_1 (AMCL + planner + controller)
  • robot-nav-robot-2: Nav2 full navigation stack for robot_2 (AMCL + planner + controller)
  • Communication: zenoh-bridge-ros2dds sidecars bridge namespaced ROS2 topics between pods over TCP

Architecture

Pod: gazebo-sim                       Pod: robot-nav-robot-1
  ├── Gazebo server (physics)           ├── AMCL (localization /robot_1/*)
  ├── spawn robot_1 at (-2, -0.5)      ├── planner_server, controller_server
  ├── spawn robot_2 at (2, 0.5)        ├── bt_navigator, map_server
  ├── Xvfb + x11vnc + noVNC            └── zenoh-bridge (peer → gazebo:7447)
  └── zenoh-bridge (router :7447)
                                        Pod: robot-nav-robot-2
  Topics bridged (examples):             ├── AMCL (localization /robot_2/*)
  /robot_1/scan  → robot-nav-robot-1     ├── planner_server, controller_server
  /robot_2/scan  → robot-nav-robot-2     ├── bt_navigator, map_server
  /robot_1/cmd_vel ← robot-nav-robot-1   └── zenoh-bridge (peer → gazebo:7447)
  /robot_2/cmd_vel ← robot-nav-robot-2

Verify Communication

# Check robot_1 Nav2 receives laser scan from Gazebo
oc exec deployment/robot-nav-robot-1 -c nav2 -- bash -c \
  'source /usr/lib64/ros-jazzy/setup.bash; timeout 5 ros2 topic echo /robot_1/scan --once'

# Send a velocity command to robot_2
oc exec deployment/robot-nav-robot-2 -c nav2 -- bash -c \
  'source /usr/lib64/ros-jazzy/setup.bash;
   ros2 topic pub /robot_2/cmd_vel geometry_msgs/msg/Twist \
     "{linear: {x: 0.2}}" -t 5'
